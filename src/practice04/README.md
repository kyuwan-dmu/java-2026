# Practice 04 — 주문이 계속 들어온다

## 상황
오전에 74건을 정리해뒀는데, 점심 지나니 신규 주문이 계속 들어옵니다.
사수가 말합니다.

> "배열은 크기가 고정이라 불편하지? ArrayList 써봐. 추가/삭제가 자유로워."

## 목표
- `ArrayList<Order>`를 선언하고 사용할 수 있다
- 배열과 리스트의 차이를 이해한다 (고정 크기 vs 가변 크기)
- `add()`, `remove()`, `get()`, `size()` 메서드를 사용할 수 있다

## 요구사항
1. `OrderDataProvider.getOrdersAsList()`로 주문 리스트를 받아오시오
2. 전체 주문 건수를 출력하시오
3. 신규 주문 2건을 직접 만들어서 리스트에 추가하시오
   - 예: `new Order("9999999", "테스트 상품", "", 1, "머지", 15000, "2024-08-28 09:00:00", "301 배송 준비 중", "서울 강남구")`
4. 추가 후 건수를 다시 출력하시오
5. 첫 번째 주문을 삭제하고 건수를 출력하시오
6. 리스트를 for-each로 순회하여 전체 출력하시오

## 출력 예시
```
현재 주문 건수: 74건

[신규 주문 추가]
추가 후 건수: 76건

[첫 번째 주문 삭제]
삭제 후 건수: 75건

===== 전체 주문 =====
[바디홀릭] 스테이누디 디스커버리 세트 (20ml*6종) - 20,000원
...
[머지] 테스트 상품 - 15,000원
====================
```

## 힌트
- `List<Order> orders = OrderDataProvider.getOrdersAsList();`
- 추가: `orders.add(new Order(...))`
- 삭제: `orders.remove(0)` (인덱스 0번 삭제)
- 크기: `orders.size()` (배열의 `.length`와 다릅니다!)
- `import java.util.List;`를 잊지 마세요
